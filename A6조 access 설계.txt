1. 테이블 설계
1.1. Movies 테이블

Fields:
	MovieID (Primary Key)
	Title (영화 제목)
	Genre (장르)
	Duration (상영 시간)
	ReleaseDate (개봉일)
	Rating (관람 등급)
1.2. Seats 테이블

Fields:
	SeatID (Primary Key)
	SeatNumber (좌석 번호)
	Status (예약 상태: 예약 가능, 예약됨)
	ScreenID (외래키, 상영관 ID)
1.3. Screens 테이블

Fields:
	ScreenID (Primary Key)
	ScreenName (상영관 이름)
	TotalSeats (총 좌석 수)
1.4. Reservations 테이블

Fields:
	ReservationID (Primary Key)
	MovieID (외래키)
	SeatID (외래키)
	CustomerName (고객 이름)
	ReservationDate (예약 날짜)

2. 쿼리 설계
	2.1. 예약 가능한 좌석 조회 쿼리
	2.2. 특정 영화의 상영 시간 조회 쿼리

3. 폼 설계
	3.1. 영화 목록 폼
		기능: 영화 목록을 보여주고, 선택한 영화의 상영 시간으로 이동할 수 있는 버튼 추가
	3.2. 좌석 선택 폼
		기능: 선택한 영화와 상영관에 따라 예약 가능한 좌석을 표시
		구성 요소:
		좌석 선택 버튼 (예약 가능 좌석)
		고객 이름 입력 필드
		예약 버튼
	3.3. 예약 확인 폼

		기능: 예약 내용을 확인하고 최종 예약할 수 있는 폼
		구성 요소:
		선택한 영화, 좌석, 고객 이름 표시
		예약 확인 버튼
